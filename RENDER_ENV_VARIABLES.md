# Render Environment Variables - Copy & Paste Guide

## Required Variables (Must Set)

### 1. ADMIN_SEED_PASSWORD
**Value:** `Admin@123`
**Description:** Password for the admin account (admin@rentalpro.et)
**Required:** YES

### 2. CORS_ALLOWED_ORIGINS
**Value:** Leave EMPTY for now (will update after frontend deployment)
**After frontend deploy, set to:** `https://your-frontend-url.vercel.app`
**Description:** Frontend URL that can access the backend API
**Required:** YES (but can be empty initially, update after frontend deploy)
**Important:** Without this, frontend CANNOT communicate with backend in production

---

## Required for Full Functionality (File Uploads)

### 3. CLOUDINARY_CLOUD_NAME
**Value:** Your Cloudinary cloud name (e.g., `dxxxxx`)
**Description:** Cloudinary cloud name for uploading documents
**Required:** YES (for profile docs, property title deeds, contract documents)
**Where to get:** [Cloudinary Dashboard](https://cloudinary.com/console) → Account Details

### 4. CLOUDINARY_API_KEY
**Value:** Your Cloudinary API key (e.g., `123456789012345`)
**Description:** Cloudinary API key for authentication
**Required:** YES (for file uploads to work)
**Where to get:** [Cloudinary Dashboard](https://cloudinary.com/console) → Account Details

### 5. CLOUDINARY_API_SECRET
**Value:** Your Cloudinary API secret (e.g., `abcdefghijklmnopqrstuvwxyz`)
**Description:** Cloudinary API secret for authentication
**Required:** YES (for file uploads to work)
**Where to get:** [Cloudinary Dashboard](https://cloudinary.com/console) → Account Details

### 6. CLOUDINARY_ENABLED
**Value:** `true` (change from default `false`)
**Description:** Enable Cloudinary file upload service
**Required:** YES (to enable file uploads)
**Note:** Already set to `false` in render.yaml - you MUST change it to `true`

---

## Auto-Generated Variables (Don't Touch)

These are automatically set by Render:

- **JWT_SECRET** - Auto-generated by Render
- **DATABASE_URL** - Auto-linked from PostgreSQL database
- **SPRING_PROFILES_ACTIVE** - Already set to `prod`
- **CLOUDINARY_ENABLED** - Already set to `false`
- **ADMIN_SEED_EMAIL** - Already set to `admin@rentalpro.et`
- **ADMIN_SEED_PHONE** - Already set to `+251900000000`

## Deployment Steps

### Step 1: Initial Deployment
1. Set only: **ADMIN_SEED_PASSWORD** = `Admin@123`
2. Leave **CORS_ALLOWED_ORIGINS** empty
3. Skip all Cloudinary variables
4. Click "Apply" to deploy

### Step 2: After Frontend Deployment
1. Deploy frontend to Vercel
2. Get frontend URL (e.g., `https://rentalpro-et.vercel.app`)
3. Go back to Render → Backend → Environment
4. Update **CORS_ALLOWED_ORIGINS** = `https://rentalpro-et.vercel.app`
5. Save (will auto-redeploy)

---

## Why CORS_ALLOWED_ORIGINS is Critical

**Without it:** Frontend will get CORS errors and cannot call backend APIs
**With it:** Frontend can successfully communicate with backend

The backend currently only allows `localhost` origins. In production, you MUST add your Vercel frontend URL to allow cross-origin requests.

---

## Summary: What to Enter Now

When Render asks for environment variables:

```
ADMIN_SEED_PASSWORD = Admin@123
CORS_ALLOWED_ORIGINS = (leave empty, update later)
CLOUDINARY_ENABLED = true
CLOUDINARY_CLOUD_NAME = (your cloudinary cloud name)
CLOUDINARY_API_KEY = (your cloudinary api key)
CLOUDINARY_API_SECRET = (your cloudinary api secret)
```

**Get Cloudinary credentials:**
1. Go to [Cloudinary](https://cloudinary.com/)
2. Sign up for free account (if you don't have one)
3. Dashboard → Account Details
4. Copy: Cloud Name, API Key, API Secret

**Without Cloudinary:** Users cannot upload profile documents, property title deeds, or contract documents.
